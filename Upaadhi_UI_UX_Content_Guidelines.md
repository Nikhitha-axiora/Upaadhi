# Upaadhi UI/UX And Content Design Guidelines

## 1. Design Objective

Upaadhi should feel simple, trustworthy, local, and income-focused.

The design goal is not to look like a corporate job portal. The app should feel like a friendly local earning companion where users can quickly find work, post work, contact people, and feel safe.

Primary product promise:

> Find nearby work. Post nearby work. Earn today.

The interface must support users who may:

- Use low-end Android phones.
- Prefer local language and simple words.
- Have limited patience for long forms.
- Prefer calling over formal job applications.
- Be worried about fake jobs, scams, and safety.
- Need confidence, speed, and clarity more than decorative design.

## 2. Core UX Principles

### 2.1 Speed First

Critical UX targets:

- Find an earning opportunity in 5 seconds.
- Post a listing in 30 seconds.
- Contact a listing owner in 1 tap.

Design rules:

- Keep the feed as the first screen after login/location.
- Avoid complex dashboards for earners.
- Avoid long onboarding.
- Use progressive profile completion instead of forcing everything upfront.

### 2.2 Trust Must Be Visible

Users must quickly understand:

- Who posted this?
- Is this person verified?
- How far is this opportunity?
- What is the pay/price?
- Is this urgent?
- Can I call safely?
- Can I report if something is wrong?

Trust signals should be visible on cards and details:

- Phone verified.
- Employer verified.
- Rating.
- Completed work count.
- Response time.
- Joined date.
- Report-safe action.

### 2.3 Friendly, Not Childish

Upaadhi should feel warm and approachable, but not toy-like.

Use:

- Soft light backgrounds.
- Clear cards.
- Human language.
- Calm colors.
- Rounded but not overly bubbly UI.
- Helpful empty states.

Avoid:

- Dark-heavy interface.
- Overuse of gradients.
- Loud red/orange everywhere.
- Complex financial language.
- Corporate HR jargon.
- Manipulative urgency.

### 2.4 Local And Practical

The UI should reflect how Indian users actually find work:

- Nearby distance.
- Call button.
- WhatsApp-like comfort.
- Clear pay.
- Simple availability.
- Local language.
- Shop/locality-level context.

Do not make the product feel like a resume portal.

## 3. Visual Identity Direction

### 3.1 Brand Feeling

Upaadhi should feel:

- Helpful.
- Fast.
- Local.
- Safe.
- Energetic.
- Practical.
- Respectful.
- Hopeful without overpromising.

### 3.2 Suggested Light Theme Palette

Use a warm, clean light theme with green as the earning/trust anchor.

Primary colors:

- Primary Green: `#1E8E5A`
- Primary Green Dark: `#136C43`
- Primary Green Soft: `#E8F6EF`

Secondary colors:

- Friendly Blue: `#2563EB`
- Blue Soft: `#EAF1FF`
- Amber: `#F59E0B`
- Amber Soft: `#FFF4D8`

Neutral colors:

- App Background: `#F7FAF8`
- Surface/Card: `#FFFFFF`
- Surface Muted: `#F1F5F3`
- Border: `#DDE7E1`
- Text Primary: `#102018`
- Text Secondary: `#516158`
- Text Muted: `#7B8A82`

Status colors:

- Success: `#16A34A`
- Warning: `#F59E0B`
- Error: `#DC2626`
- Info: `#2563EB`

### 3.3 Category Colors

Use category colors consistently, but keep them subtle.

Jobs:

- Badge: Green `#1E8E5A`
- Soft background: `#E8F6EF`

Services:

- Badge: Blue `#2563EB`
- Soft background: `#EAF1FF`

Sell:

- Badge: Amber `#F59E0B`
- Soft background: `#FFF4D8`

Rent:

- Badge: Violet `#7C3AED`
- Soft background: `#F1EAFE`

Safety/Verification:

- Badge: Teal `#0F766E`
- Soft background: `#E6F5F3`

### 3.4 Color Psychology Use

Green:

- Use for earning, success, verification, trust, and primary action.

Blue:

- Use for services, information, and stable guidance.

Amber:

- Use for urgency, attention, and pending states.

Red:

- Use only for errors, danger, fraud alerts, destructive actions, and safety warnings.

Violet:

- Use sparingly for rent/premium/future AI features.

Rule:

- The app should not look like a warning system. Red should be rare.

## 4. Typography Guidelines

### 4.1 Font Direction

Use a clean, highly readable sans-serif.

Recommended:

- Android/system: Roboto.
- Web/admin: Inter or system font.
- Indian language support: Noto Sans family.

### 4.2 Type Scale

Mobile:

- Page title: 22-24 px.
- Section title: 18-20 px.
- Card title: 16-18 px.
- Body: 14-16 px.
- Metadata: 12-13 px.
- Button text: 14-16 px.

Rules:

- Do not use tiny text for pay, distance, or safety details.
- Keep line height comfortable.
- Avoid all caps except very small labels/badges.
- Avoid negative letter spacing.

## 5. Layout System

### 5.1 Spacing

Use an 8 px spacing system.

Recommended:

- Screen horizontal padding: 16 px.
- Card padding: 12-16 px.
- Section gap: 16-24 px.
- Small element gap: 6-8 px.
- Button height: 44-48 px.

### 5.2 Cards

Listing cards should be simple and scannable.

Card rules:

- White surface.
- 1 px soft border.
- 8 px border radius.
- Minimal shadow, if any.
- Stable height where possible.
- No nested cards.

Card must show:

- Category badge.
- Title.
- Pay/price.
- Distance/locality.
- Posted time.
- Trust signal.
- Primary action.

### 5.3 Bottom Navigation

Use 4 primary tabs:

- Feed.
- Post.
- Chats or Activity.
- Profile.

Avoid:

- Too many tabs.
- Separate tabs for Jobs, Services, Sell, Rent.

Categories should be filters, not main navigation.

## 6. Psychometric And Behavioral Design Patterns

These patterns should build confidence, reduce anxiety, and help users act. They must not exploit fear or mislead users.

### 6.1 Cognitive Ease

Users should understand a screen within 2 seconds.

Use:

- Familiar icons.
- Short labels.
- Clear pay/distance.
- Repeated card structure.
- Predictable navigation.

Avoid:

- Multiple competing CTAs.
- Long paragraphs.
- Dense forms.
- Hidden actions.

### 6.2 Social Proof

People trust activity.

Use:

- "12 people contacted today" only if true.
- "Verified employer."
- "24 completed works."
- "Usually replies in 10 min."

Avoid:

- Fake counters.
- Fake urgency.
- Inflated success claims.

### 6.3 Commitment Gradient

Start with small actions before asking for more effort.

Example:

1. Select location.
2. Browse feed.
3. Tap call/chat.
4. Complete profile later.

For posting:

1. Choose listing type.
2. Add title/pay/location.
3. Publish.
4. Improve listing later.

### 6.4 Loss Aversion, Used Ethically

Users respond to missed opportunities, but the app should not scare them.

Good:

- "This job is for today."
- "5 km away."
- "Closes after 6 PM."

Bad:

- "You will lose income if you do not apply now."
- Fake countdowns.
- Artificial scarcity.

### 6.5 Trust Through Control

Users feel safer when they have control.

Provide:

- Report.
- Block.
- Hide phone until user chooses.
- Exact location privacy.
- Notification controls.
- Delete listing.
- Pause listing.

### 6.6 Reward And Progress

Use progress to motivate profile completion.

Examples:

- "Profile 60% complete."
- "Add skills to get better matches."
- "Verify employer profile to build trust."

Avoid:

- Shame language.
- Blocking key usage too early.

### 6.7 Local Belonging

Use locality to create relevance.

Examples:

- "Work near Kukatpally."
- "Hiring within 3 km."
- "Services near Indiranagar."

Users should feel the app knows their area, not just their city.

### 6.8 Safety Friction

Add friction only for risky actions.

Low friction:

- Browse feed.
- Contact verified employer.
- Post low-risk service.

More friction:

- High-value rental.
- Night job.
- Cash collection job.
- Employer asking upfront fee.
- High-value electronics sale.

## 7. Screen-Level Guidelines

### 7.1 Onboarding

Goal:

- Get user into feed quickly.

Flow:

1. Phone number.
2. OTP.
3. Location.
4. Role preference optional.
5. Feed.

Do not force:

- Resume.
- Full profile.
- ID verification.
- Long skill selection.

Content:

- "Enter your mobile number"
- "We will send a verification code"
- "Choose your work location"
- "You can change this anytime"

### 7.2 Home Feed

Feed should be the heart of the app.

Top area:

- Location selector.
- Search bar.
- Filter chips.

Filter chips:

- Nearby.
- Jobs.
- Services.
- Sell.
- Rent.
- Today.
- Verified.

Card action:

- Jobs: Call / Chat.
- Services: Chat / Call.
- Sell: Chat / Call.
- Rent: Chat / Call.

### 7.3 Listing Card

Recommended structure:

- Top row: category badge + posted time.
- Main title.
- Pay/price line.
- Location and distance.
- Trust row.
- CTA row.

Example:

```text
Job · Today
Helper needed at grocery shop
Rs 600/day
1.8 km · Ameerpet
Verified phone · Replies fast
[Call] [Chat]
```

### 7.4 Listing Detail

Must answer:

- What is this?
- How much can I earn/pay?
- Where is it?
- Who posted it?
- Can I trust them?
- What action can I take?

Sections:

- Title and category.
- Pay/price.
- Location.
- Description.
- Requirements/details.
- Poster profile.
- Safety notes.
- Similar listings.

Sticky bottom action:

- Call.
- Chat.

### 7.5 Post Flow

Goal:

- Publish in under 30 seconds.

Steps:

1. Choose type.
2. Add required details.
3. Add location.
4. Preview.
5. Publish.

Use dynamic forms:

- Job fields for jobs.
- Service fields for services.
- Product fields for selling.
- Rental fields for renting.

Do not show all fields for all categories.

### 7.6 Profile

Profile should build trust, not vanity.

Show:

- Name.
- Photo.
- Locality.
- Roles.
- Skills.
- Rating.
- Completed work.
- Verification badges.
- Active listings.
- Response time.

Profile completion prompt:

- "Add 2 skills to improve your matches."
- "Verify your employer profile to get more responses."

### 7.7 Admin Panel

Admin design should be efficient and calm.

Priorities:

- Safety queue.
- Report queue.
- Listing review.
- User lookup.
- Moderation action.
- Audit history.

Admin UI should be dense but readable.

Avoid:

- Marketing-style dashboards.
- Decorative cards.
- Hidden moderation actions.

## 8. Content Design Guidelines

### 8.1 Voice And Tone

Tone:

- Simple.
- Respectful.
- Helpful.
- Direct.
- Local.
- Encouraging.

Avoid:

- Corporate HR language.
- English-heavy complex words.
- Fear-based messaging.
- Overpromising guaranteed income.

### 8.2 Writing Rules

Use:

- Short sentences.
- Simple verbs.
- Clear labels.
- Action-first copy.

Avoid:

- "Opportunities curated for your professional aspiration."
- "AI-powered economic empowerment ecosystem."
- "Submit candidature."

Prefer:

- "Find work near you."
- "Call employer."
- "Post a job."
- "Add pay."
- "Report this listing."

### 8.3 Button Labels

Use clear action labels:

- Call.
- Chat.
- Apply.
- Post.
- Publish.
- Save.
- Share.
- Report.
- Block.
- Verify.
- Close listing.

Avoid vague labels:

- Proceed.
- Continue, when the next action is unclear.
- Submit, unless form context is obvious.

### 8.4 Empty States

Empty states should guide action.

No listings nearby:

- "No listings nearby yet."
- "Try increasing distance or changing category."

No chats:

- "No chats yet."
- "Contact a listing to start."

No posted listings:

- "You have not posted anything yet."
- "Post work, service, product, or rent item."

### 8.5 Safety Copy

Use calm safety reminders.

Examples:

- "Never pay money to get a job."
- "Meet in a safe public place for product exchange."
- "Check details before accepting night work."
- "Report if someone asks for registration fees."

Avoid:

- Scary warnings everywhere.
- Blaming users.
- Legal-heavy language in normal flows.

## 9. Iconography Guidelines

Use familiar icons.

Recommended:

- Home/feed.
- Plus/post.
- Message/chat.
- Phone/call.
- User/profile.
- Shield/verified.
- Map pin/location.
- Filter.
- Search.
- Star/rating.
- Flag/report.
- Ban/block.

Rules:

- Icons should support labels, not replace critical text in early versions.
- Use tooltips on web/admin.
- Keep icon style consistent.

## 10. Form Design Guidelines

Forms must be short.

Rules:

- Use step-by-step posting.
- Show only required fields first.
- Use defaults.
- Use chips for common options.
- Use numeric keypad for pay/price.
- Use location picker/manual locality entry.
- Save draft automatically if possible.

Validation:

- Inline.
- Clear.
- Human.

Examples:

- "Add pay amount."
- "Choose a location."
- "Title should be at least 5 characters."

## 11. Accessibility Guidelines

Requirements:

- Minimum 44 px tap targets.
- Strong contrast.
- No critical meaning by color alone.
- Text should scale without breaking layout.
- Buttons must not truncate important labels.
- Support screen reader labels for important actions.
- Avoid tiny metadata.

## 12. Localization Guidelines

Prepare for:

- English.
- Hindi.
- Telugu.
- Tamil.
- Kannada.
- Malayalam.
- Marathi.

Rules:

- Keep strings externalized.
- Avoid hardcoded text in UI.
- Avoid layouts that break with longer local-language text.
- Use simple vocabulary.
- Test key flows in pilot language.

## 13. Notification Content

Notification rules:

- Useful.
- Timely.
- Not spammy.
- Local and specific.

Examples:

- "New helper job near Ameerpet."
- "Your listing is live."
- "Ravi sent you a message."
- "Your report was received."

Avoid:

- "Amazing opportunity waiting for you!!!"
- Too many daily promotional notifications.
- Fake urgency.

## 14. Design System Components

Core components:

- App header.
- Location selector.
- Search bar.
- Filter chips.
- Listing card.
- Category badge.
- Trust badge.
- Primary button.
- Secondary button.
- Icon button.
- Bottom navigation.
- Step form.
- Text input.
- Price/pay input.
- Distance selector.
- Rating display.
- Report sheet.
- Confirmation modal.
- Toast/snackbar.
- Empty state.
- Loading skeleton.

## 15. Motion And Feedback

Use motion lightly.

Good:

- Button press feedback.
- Skeleton loaders.
- Smooth bottom sheets.
- Success confirmation.

Avoid:

- Heavy animations.
- Confetti for serious work actions.
- Motion that delays contact/posting.

## 16. Do And Do Not

Do:

- Keep feed first.
- Make pay and distance visible.
- Use one unified listing card.
- Use category colors consistently.
- Make report/block easy.
- Keep posting fast.
- Use friendly local copy.
- Build trust signals into every important screen.

Do not:

- Create separate apps inside the app.
- Hide contact actions.
- Force resume upload.
- Force verification before basic browsing.
- Use fake urgency.
- Use too much red.
- Over-market AI.
- Make workers pay early.

## 17. MVP Design Checklist

Before MVP launch:

- Feed is first screen after login/location.
- Listing cards show pay, distance, category, and trust.
- Posting flow can be completed in under 30 seconds.
- Call/chat action is visible.
- Report/block is available.
- Exact location is not exposed by default.
- Empty states guide next action.
- Text works in selected pilot language.
- Light theme contrast passes basic accessibility checks.
- Forms are usable on low-end Android screens.
- No screen feels like a corporate job portal.

## 18. Final Design Direction

Upaadhi should feel like a trusted local earning app.

The design should create a feeling of:

- "I can understand this."
- "This is near me."
- "This looks safe enough to contact."
- "I can earn today."
- "I can post without stress."

If the UI achieves that, Upaadhi will feel different from job portals, classifieds, and informal WhatsApp groups.
